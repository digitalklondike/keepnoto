# Keepnoto Design Direction

Keepnoto should feel like a modern personal knowledge vault: warm, calm, premium, simple, and useful.

## Visual Direction

- Modern.
- Warm.
- Calm.
- Premium but simple.
- Soft gradients.
- Bento cards.
- Strong typography hierarchy.
- Clean spacing.
- Personal knowledge vault feeling.

## Saved Link Experience

Saved links should feel valuable, not like a boring bookmark list. Cards should make it easy to understand what a saved item is, where it came from, and why it matters.

Prioritize:

- Clear saved-link titles.
- Prominent "why saved" context.
- Useful metadata such as source, collection, tags, or date.
- Calm card layouts with enough breathing room.
- Empty states that help users start saving links with context.

## Avoid

- Generic blue SaaS.
- Childish UI.
- Boring bookmark-manager visuals.
- Overloaded dashboards.
- Weak typography.
- Low contrast text.
- Too many borders.
- Noisy cards.
- Random glassmorphism.

## UI Review Standard

Before finishing UI work, check that the screen feels calm, organized, responsive, readable, and specific to Keepnoto. The result should help users feel that their saved links are understandable and worth returning to.
