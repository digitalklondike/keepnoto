# Keepnoto MVP Scope

The MVP should prove that Keepnoto can help users save links with context and find them again later.

## In Scope

- Saving a link.
- Adding a "why saved" note or reason.
- Optional title and description.
- Collections or folders.
- Tags.
- Search.
- Filtering.
- Clean dashboard or home page.
- Saved link cards.
- Add link flow.
- Empty states.
- Basic responsive layout.

## MVP Authentication

- Google OAuth is the only sign-in method for this MVP.
- Email Magic Links, one-time email codes, Resend, and custom SMTP are deferred until a verified product domain is available.
## Out Of Scope

- AI summaries.
- Browser extension.
- Team collaboration.
- Public sharing.
- Payments.
- Mobile app.
- Complex automations.
- Recommendations.
- Integrations.

## Planning Guidance

- Prioritize the save flow, saved-link card quality, search, and simple organization.
- Treat the "why saved" note as a first-class field, not optional decoration.
- Prefer small, complete user workflows over broad dashboard complexity.
- Do not add advanced features until the core save-and-return loop feels strong.
