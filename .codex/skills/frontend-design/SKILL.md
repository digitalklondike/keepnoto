# Frontend Design Skill for Keepnoto

Use this skill when creating or improving UI for Keepnoto.

## Product feeling

Keepnoto should feel:
- warm
- calm
- personal
- useful
- modern, but not generic
- more like a thoughtful personal library than a corporate SaaS dashboard

Avoid:
- generic AI-looking landing pages
- random glassmorphism
- meaningless gradients
- fake SaaS blocks
- overdesigned hero sections
- too many cards without purpose

## UI rules

- Prefer clean layouts with clear visual hierarchy.
- Use enough spacing; do not make the UI cramped.
- Use warm, soft backgrounds and subtle surfaces.
- Use readable typography.
- Keep components reusable.
- Prefer product screens over marketing fluff.
- Make saved links feel contextual, not just bookmarked.
- The “why I saved this” field is important and should be visually highlighted.
- Do not add new dependencies unless clearly needed.
- Do not change the product direction without asking.
